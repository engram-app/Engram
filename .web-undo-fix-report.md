# Web CRDT Undo Fix Report

## Root Cause Restated

`buildEditorState` installed both `history()` (the `@codemirror/commands` native offset-based undo extension) and `keymap.of([...defaultKeymap, ...historyKeymap])`. The `historyKeymap` binds `Ctrl+Z` to the native `undo` command.

`yCollab` also installs its own `Y.UndoManager` internally (via `yUndoManagerFacet` and `yUndoManager`), but the `yUndoManagerKeymap` was NOT installed, so `Ctrl+Z` always fired the native `undo` command instead.

Native undo operates on raw CodeMirror offset-based transactions. When a remote peer's change arrives, `ySync` forwards it into the CM view as a regular transaction, which lands in the native history stack. Pressing `Ctrl+Z` pops that remote transaction from the native history and dispatches a delete back through `yCollab` into the shared `Y.Text` -- which then propagates the deletion to all other peers as if it were a local edit. This is the divergence: the local client reverts a change made by a remote peer.

`Y.UndoManager` avoids this by tracking only transactions originating from the local `ySyncFacet` config, ignoring remote-origin changes. It correctly undoes only the local user's edits.

## The Fix

In `frontend/src/viewer/note-editor.tsx`:

1. Removed `history()` from extensions (no more native history field).
2. Removed `historyKeymap` from the keymap list.
3. Dropped `history` and `historyKeymap` from the `@codemirror/commands` import (only `defaultKeymap` remains).
4. Imported `yUndoManagerKeymap` from `y-codemirror.next`.
5. Installed `Prec.highest(keymap.of(yUndoManagerKeymap))` so `Ctrl+Z` / `Cmd+Z` / redo are handled by `Y.UndoManager` at the highest keymap priority, before `defaultKeymap`.

`yCollab` already creates and installs a `Y.UndoManager` by default -- the only missing piece was the keymap.

## Test Approach

Two new tests were added to `src/viewer/note-editor.test.tsx`:

**1. Structural test (`does not install the native history field`):**
Builds an `EditorState` and asserts `state.field(historyField, false)` is `undefined`. `historyField` is the `StateField` that `history()` registers; its absence proves native undo is not wired. This test is simple, fast (no DOM), and precisely targets the extension-list regression.

**2. Behavioral test (`Ctrl+Z reverts only the local edit and preserves remote peers text`):**
Mounts a real `EditorView` in happy-dom (the earlier comment in the test file said this was impossible; it was overly cautious -- the DOM stubs in `test-setup.ts` are sufficient). The test:
- Dispatches a local insert transaction (`L`) via `view.dispatch`, which `ySync` forwards into the `Y.Text` with `syncConf` origin (tracked by `Y.UndoManager`).
- Inserts a remote string (`R`) directly via `doc.transact(() => ytext.insert(...), 'remote-peer')`, which `ySync` picks up but `Y.UndoManager` ignores (foreign origin).
- Simulates `Ctrl+Z` via `runScopeHandlers(view, ctrlZ, 'editor')` -- the same dispatch path as a real keypress, so priority between `historyKeymap` and `yUndoManagerKeymap` is actually exercised.
- Asserts: `R` still present, `L` gone, `view.state.doc.toString() === ytext.toString()` (no divergence).

No jsdom fallback was needed; full behavioral testing was feasible.
